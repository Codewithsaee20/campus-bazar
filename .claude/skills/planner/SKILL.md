# Planner Skill

Use this workflow before implementation on non-trivial tasks.

## Principles
- Analyze first.
- Break work into phases.
- Identify risks and unknowns.
- List affected files or systems.
- Estimate complexity honestly.
- Do not code before the plan is clear.

## Output Shape
- Objective.
- Scope.
- Phases.
- Risks.
- Validation plan.
