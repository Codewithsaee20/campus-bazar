# /summarize

Goal: provide a compact snapshot of the repository or working area.

Workflow:
1. Identify the most relevant changed files or active areas.
2. Summarize behavior, intent, and notable risks.
3. Keep the response dense and avoid reprinting code.

Output:
- Current state in a few bullets.
- Important files or modules.
- Open issues, risks, or next actions.
