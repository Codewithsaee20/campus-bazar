# /deploy

Goal: confirm the project is ready to ship.

Workflow:
1. Check the relevant build, test, lint, and packaging steps.
2. Look for environment, configuration, and release risks.
3. Confirm the change set is coherent and minimal.
4. Report blockers before recommending release.

Output:
- Deployment readiness verdict.
- Blocking issues first.
- Validation steps completed.
- Remaining release risks.
