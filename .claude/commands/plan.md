# /plan

Goal: turn a request into a practical implementation plan.

Workflow:
1. Restate the objective in engineering terms.
2. Identify the likely files, modules, or layers involved.
3. Break the work into phases with dependencies.
4. Call out risks, unknowns, and validation strategy.

Output:
- No code.
- Clear phases and sequencing.
- Affected files or systems.
- Risks, assumptions, and estimated complexity.
